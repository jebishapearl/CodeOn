Find Second Maximum Element in an Array

Problem Statement

Given an array of integers, find the second maximum element in the array.

Example 1

Input:
5
1 3 7 2 9

Output:
7

Explanation:
The maximum element in the array is 9, and the second maximum element is 7. Hence, the output is 7.

Example 2

Input:
4
10 5 15 20

Output:
15

Explanation:
The maximum element in the array is 20, and the second maximum element is 15. Hence, the output is 15.

Constraints

  • 1 ‚â§ n ‚â§ 10‚Å∂ (number of elements in the array)
  • 0 ‚â§ array elements ‚â§ 10‚Å∂ (elements of the array)

Starter Code

Copy the template into the IDE

#include 

int main() {
    int n;
    scanf("%d", &n);

    int arr[n];
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }

    int max = arr[0], second_max = -1;
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) {
            second_max = max;
            max = arr[i];
        } else if (arr[i] > second_max && arr[i] != max) {
            second_max = arr[i];
        }
    }

    printf("%d\n", second_max);
    return 0;
}

Explanation

üìã Problem Summary

You are given an array of integers. Your goal is to find the second largest element ‚Äî that is, the number which is smaller than the maximum, but larger than all others.

üî¢ Input Format

  • First line: Integer n ‚Äî number of elements.
  • Second line: n space-separated non-negative integers.

üß≠ Approach

  1. Initialize two variables: max and second_max.
  2. Iterate through the array.
  3. If the current number is greater than max, update both max and second_max.
  4. If the current number is less than max but greater than second_max, update second_max.
  5. After the loop ends, check if a valid second_max was found.

‚ö†Ô∏è Note for Advanced Learners

In this version of the problem, all numbers are non-negative, so initializing second_max to -1 works safely.

However, if negative numbers were allowed, using -1 would cause issues. For example, what if the array only had values like -5, -10, or -100? In such cases, -1 would incorrectly remain the highest.

To handle such scenarios correctly, you'd use:

#include <limits.h>
int second_max = INT_MIN;

Here, INT_MIN represents the smallest value that can be stored in an integer. It's defined inside the <limits.h> library, which gives us standardized constants like INT_MIN and INT_MAX.

But since this is a beginner-friendly logic-building exercise and all inputs are guaranteed to be non-negative, we‚Äôve kept things simple and skipped importing extra headers.

‚ö†Ô∏è Edge Cases

  • All elements are the same ‚Üí No second maximum.
  • Array size is 1 ‚Üí No second maximum.

  • Space: O(1)
