Reverse an Array

Problem Statement

Given an array of integers, reverse the array.

Example

Input:

        5
        1 3 7 2 9

Output:

        9 2 7 3 1

Explanation:

The array is reversed from [1, 3, 7, 2, 9] to [9, 2, 7, 3, 1].

Starter Code

Copy the template into the IDE

#include 

int main() {
    int n;
    scanf("%d", &n);

    int arr[n];
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }

    // Reversing the array
    for (int i = 0; i < n / 2; i++) {
        int temp = arr[i];
        arr[i] = arr[n - i - 1];
        arr[n - i - 1] = temp;
    }

    // Output the reversed array
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
    return 0;
}

Hints

Hint 1 🔍

Start by reading the input array. Then, use a loop to swap elements from the start and end of the array until you reach the middle.

Hint 2 🤔

Make sure to stop the loop when you reach the middle element, as it doesn’t need to be swapped.

Explanation

📋 Problem Summary

You are given an array of integers. Your task is to reverse the array, i.e., rearrange the elements in opposite order.

🔢 Input Format

  • First line: Integer n — number of elements.
  • Second line: n space-separated integers.

🧭 Approach

  1. Use two pointers: one at the beginning (i = 0) and one at the end (j = n - 1).
  2. Swap the elements at positions i and j.
  3. Move i forward and j backward.
  4. Repeat this process until i is greater than or equal to j.

✅ Example Dry Run

Input: 5
1 3 7 2 9

Step-by-step:
Initial: [1, 3, 7, 2, 9]
Swap arr[0] and arr[4] → [9, 3, 7, 2, 1]
Swap arr[1] and arr[3] → [9, 2, 7, 3, 1]
arr[2] is the middle, no need to swap.

Output: 9 2 7 3 1

⚠️ Edge Cases

  • Single element array → Output is the same.
  • All elements are the same → Reversing makes no difference.
  • Empty array → Nothing to reverse.

🧠 Complexity

  • Time: O(n/2) → simplifies to O(n)
  • Space: O(1) → in-place reversal
