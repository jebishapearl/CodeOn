Count Odd Numbers in an Array

Problem Statement

Given an array of integers, count the number of odd numbers in the array.

Example 1

Input:
5
1 2 3 4 5

Output:
3

Explanation:
The odd numbers are 1, 3, and 5, so the count is 3.

Example 2

Input:
4
10 11 12 13

Output:
2

Explanation:
The odd numbers are 11 and 13, so the count is 2.

Constraints

  • 1 ≤ n ≤ 10⁶ (number of elements in the array)
  • −10⁶ ≤ array elements ≤ 10⁶ (elements of the array)

Try It Yourself

Solution 
#include <stdio.h>
int main() {
    int n, count = 0;
    printf("Enter the number of elements: ");
    scanf("%d", &n);

    int arr[n];
    printf("Enter the elements: ");
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }

    for (int i = 0; i < n; i++) {
        if (arr[i] % 2 != 0) {
            count++;
        }
    }

    printf("Count of odd numbers: %d\n", count);
    return 0;
}
Explanation

🧾 Problem Understanding

You are given an array of integers. Your goal is to count how many of them are odd. A number is odd if it is not divisible by 2 — in other words, number % 2 != 0.

🔢 Input Format

  • The first line contains an integer n — the number of elements in the array.
  • The second line contains n space-separated integers — the array elements.

💡 Step-by-Step Approach

  1. Read the number n.
  2. Read n integers into an array.
  3. Initialize a variable count to 0.
  4. Loop through each element of the array.
  5. If the element is odd (arr[i] % 2 != 0), increment the count.
  6. Print the value of count.

📘 Language-Specific Notes

  • C: Use a loop with scanf and a conditional check.
  • Python: Loop using for and use if num % 2 != 0 to count.
  • Java: Use Scanner for input and arr[i] % 2 != 0 to check for odd numbers.

⚙️ Time and Space Complexity

  • Time Complexity: O(n) — Each number is checked once.
  • Space Complexity: O(n) — For storing the array.

🔍 Edge Cases to Consider

  • All numbers are odd → output = n.
  • No odd numbers → output = 0.
  • Single negative odd number → still counted.
  • Array of size 1 → always check edge limits.
