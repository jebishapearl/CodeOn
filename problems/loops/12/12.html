Check if a Number is a Palindrome

Problem Statement

Given a number N, determine whether it is a palindrome. A number is a palindrome if it remains the same when its digits are reversed.

Example 1

Input:
121

Output:
YES

Explanation:
The number 121 is a palindrome as it remains the same when its digits are reversed.

Example 2

Input:
123

Output:
NO

Explanation:
The number 123 is not a palindrome as its reverse is 321.

Explanation

View Explanation

To check if a number is a palindrome, we reverse its digits and compare the reversed number with the original.

Here's how the logic works in all three languages (C, Python, Java), using the same fundamental idea:

  1. We store the original number in a separate variable (e.g., original).
  2. Then we reverse the number using a loop:
    • Extract the last digit using % 10
    • Add it to the reversed number after shifting existing digits by one place (multiply by 10)
    • Remove the last digit using integer division /= 10
  3. Once reversed, we check: is reversed == original? If yes, it's a palindrome.

Does the logic differ between C, Python, and Java?

Nope! The approach is exactly the same in all three languages. The only differences are:

  • In C and Java, you must declare variable types (like int), and use printf/scanf or System.out.println.
  • In Python, there's no need for type declarations or semicolons. You simply use input() and print().
