Loops and Arrays - The Basics

1. Introduction to Loops

What are loops?

Loops are control structures that allow repeated execution of a block of code based on a condition.

Types of loops
  • For loop - Used when the number of iterations is known.
  • While loop - Used when the number of iterations is unknown.
  • Do-while loop - Similar to while, but executes at least once.
Key terminology
  • Initialization - Setting the loop variable.
  • Condition - The check that decides whether the loop continues.
  • Increment/Decrement - Changing the loop variable per iteration.
Examples of each loop type

1. For Loop ‚Äì Best when number of iterations is known:

for (int i = 0; i < 5; i++) {
    printf("Count: %d\n", i);
}

2. While Loop ‚Äì Use when condition must be checked before running:

int i = 0;
while (i < 5) {
    printf("Count: %d\n", i);
    i++;
}

3. Do-While Loop ‚Äì Guarantees at least one execution:

int i = 0;
do {
    printf("Count: %d\n", i);
    i++;
} while (i < 5);
Common pitfalls
  • Infinite loops: Forgetting to update the loop variable.
  • Off-by-one errors: Using <= instead of < (or vice versa).
  • Wrong loop boundaries: Starting from 1 instead of 0, or ending at n instead of n - 1.
  • Modifying loop variables inside nested logic: Can cause unexpected behavior.

Tip: Always dry-run your loop with pen and paper or use debug prints to track loop variables.

2. Introduction to Arrays

What are arrays?

Arrays are data structures that store multiple values of the same type.

Why use them?

They help organize and process collections of data efficiently.

Key terminology
  • Index - The position of an element in an array.
  • Length - The number of elements in an array.
  • Traversing - Accessing each element in an array.
Declaring and accessing arrays

1. Declaration

// Static array with size 5
int numbers[5];

// Declaration with initialization
int scores[3] = {85, 90, 78};

// Compiler infers size
int primes[] = {2, 3, 5, 7, 11};

2. Accessing Elements

int first = scores[0];   // Access first element
int last = scores[2];    // Access third element

// Modify an element
scores[1] = 95;

3. Looping Through an Array

for (int i = 0; i < 3; i++) {
    printf("Score %d: %d\n", i, scores[i]);
}

Important Notes:

  • Array indices start at 0.
  • Accessing out-of-bound indices can lead to garbage values or crashes.
  • Array size must be known at compile time in C (for static arrays).
Basic array operations
  • Summing elements
  • Finding the maximum value
Common pitfalls
  • Out-of-bounds access: Trying to access array[n] when the last valid index is n - 1.
    Example: 
    int arr[3] = {1, 2, 3};
printf("%d", arr[3]); // ‚ùå Invalid ‚Äì crashes or gives garbage
  • üîÅ Modifying an array while iterating: Can skip elements or behave unexpectedly.
    Example: 
    // Risky logic when removing items
for (int i = 0; i < n; i++) {
    if (arr[i] == 0) {
        remove(arr, i); // modifies array and size
    }
}
  • Forgetting array size: Using sizeof incorrectly on pointers.
    Example: 
    int* ptr = malloc(5 * sizeof(int));
sizeof(ptr) / sizeof(int); // ‚ùå Wrong ‚Äì gives size of pointer, not array
  • Reusing arrays without resetting values: Old values can cause bugs if not cleared.
  • Off-by-one errors: Common when using <= instead of <.
  • Using uninitialized arrays: Reading before writing can lead to unpredictable values.

‚úÖ Tip: Always track array boundaries and initialize variables before use. Use tools like Valgrind (in C/C++) to catch memory errors.

Advanced Loops and Array Techniques

1. Looping Through Arrays Efficiently

Using for, while, and for-each loops for array traversal

1. For Loop ‚Äì Best when you need the index

int arr[] = {10, 20, 30, 40, 50};
int n = sizeof(arr) / sizeof(arr[0]);

for (int i = 0; i < n; i++) {
    printf("Index %d = %d\n", i, arr[i]);
}

‚úîÔ∏è Full control over the index
‚ùå Easy to make off-by-one errors

2. While Loop ‚Äì Good when the number of elements isn't fixed ahead of time

int i = 0;
while (i < n) {
    printf("Value = %d\n", arr[i]);
    i++;
}

‚úîÔ∏è Flexible structure for dynamic conditions
‚ùå Manual index management required

3. For-Each Loop (in languages that support it)

// Example in C++ using range-based for loop
int arr[] = {10, 20, 30, 40, 50};
for (int value : arr) {
    std::cout << "Value = " << value << std::endl;
}

‚úîÔ∏è Clean syntax when index is not needed
‚ùå Not available in C (only C++) and doesn't give index access

Note: In C, for-each loops aren't available. Stick to for/while loops.

When to use each type
  • For loop - When the number of iterations is fixed.
  • While loop - When the stopping condition isn't known in advance.
Avoiding common mistakes
  • Forgetting to update loop variables (causing infinite loops).
  • Using an incorrect loop condition (off-by-one errors).

2. Nested Loops

What are nested loops?

A loop inside another loop, useful for handling 2D structures.

When to use them
  • Pattern printing
  • Matrix traversal
Examples in C and Python

üîπ Task: Print a 3x3 multiplication table using nested loops

C Version:

#include <stdio.h>

int main() {
    for (int i = 1; i <= 3; i++) {
        for (int j = 1; j <= 3; j++) {
            printf("%d\t", i * j);
        }
        printf("\n");
    }
    return 0;
}

Python Version:

for i in range(1, 4):
    for j in range(1, 4):
        print(i * j, end="\t")
    print()

‚úîÔ∏è Both Output:

1   2   3
2   4   6
3   6   9

Key Differences:

  • C requires explicit data types and main() function.
  • Python uses indentation to define loop bodies instead of braces.
  • range(1, 4) in Python includes 1 but excludes 4 ‚Äî equivalent to i = 1; i <= 3 in C.
Performance considerations

Nested loops often lead to O(n¬≤) complexity, making them inefficient for large inputs.

3. Modulo Operator for Digit Isolation

What is the modulo operator (%) and why is it useful?

The modulo operator helps extract remainders, useful in number-related problems.

Extracting digits from a number
  • Reversing digits
  • Checking if a number is a palindrome
Examples in C and Python

üîπ Task: Extract and print each digit of a number (e.g., 1234 ‚Üí 4, 3, 2, 1)

C Version:

#include <stdio.h>

int main() {
    int num = 1234;

    while (num > 0) {
        int digit = num % 10;  // Get last digit
        printf("%d\n", digit);
        num = num / 10;        // Remove last digit
    }

    return 0;
}

Python Version:

num = 1234

while num > 0:
    digit = num % 10     # Get last digit
    print(digit)
    num = num // 10      # Remove last digit

‚úîÔ∏è Output for both:

4
3
2
1

Key Notes:

  • Both use % to extract the last digit.
  • / in C performs integer division if both operands are integers.
  • Python uses // for integer division; / gives a float.
  • Digits are printed in reverse order of the original number.
Common mistakes
  • Confusing division (/) with modulo (%).
  • Incorrect order of operations when extracting digits.

4. Final Notes and Best Practices

Combining loops and arrays for efficient problem-solving

Arrays store collections of data, while loops allow us to process that data systematically. When used together, they form the foundation for solving a wide range of computational problems efficiently.

How this combination helps:

  • Sequential processing: Loops let you visit each element of an array one-by-one in a predictable, repeatable way.
  • Conditional logic: You can evaluate and respond to patterns within the data (e.g., filtering values, applying conditions) during traversal.
  • Efficiency: Linear passes through arrays (O(n) time) avoid unnecessary computations or storage, leading to optimized performance.
  • Flexibility: The same loop structure can be reused with different logic (e.g., counting, transforming, searching) depending on the problem‚Äôs goal.
  • Low space overhead: Most problems can be solved using just the loop and the array itself, without extra memory.

In short, loops handle *how* we move through data, and arrays define *what* data we're working with. Mastering their interaction is key to becoming an efficient problem-solver.

Debugging tips for loop-related errors
  • Using print statements to track loop variables.
  • Breaking loops early to check conditions.
Optimizing loops for better performance
  • Avoid unnecessary iterations.
  • Use efficient data structures alongside loops.
